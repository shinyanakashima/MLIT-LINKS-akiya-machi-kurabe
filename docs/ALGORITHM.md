# 集計アルゴリズム仕様

本ツールが、上流パイプライン（[MLIT-LINKS-akiya-pipeline](https://github.com/shinyanakashima/MLIT-LINKS-akiya-pipeline)）の
正規化済みレコードから、自治体別の集計成果物（`public/data/*.json`）をどう計算しているかをまとめます。
集計のコアは純粋関数（IO なし）で [`src/lib/aggregate-core.ts`](../src/lib/aggregate-core.ts) に、
ビニング・中央値などの補助関数は [`src/lib/buckets.ts`](../src/lib/buckets.ts) にあります。
「似た自治体」の判定など一部はUI側（[`src/pages/`](../src/pages/)）にあります。

> 前提: すべての計算は**入力に対して決定的**です（乱数なし）。同じ入力からは常に同じ成果物が得られます。

## 1. 集計単位と自治体ID

- レコードを `prefecture + " " + city` でグループ化します（`location` が欠損なら「不明」）。
- 各自治体の ID は **FNV-1a(32bit) → base36** の決定的ハッシュ（`fnv1a("県 市")`）。詳細 JSON のファイル名にも使います。
- ハッシュ衝突は稀ですが、念のため衝突時は `-1`, `-2` … を付与して一意化します。

## 2. 件数と率（母数の扱い）

| 指標 | 定義 |
| --- | --- |
| 掲載総数 `total` | グループ内のレコード数 |
| 登録 `registered` / 成約 `closed` | `status` の集計 |
| 成約率 `closed_rate` | `closed / total`（`total = 0` のときは 0） |
| PR文タグ率 `tag_rates[k]` | `タグ k が true の件数 / tagged` |

- **タグ率の分母は「全件」ではなく「タグ付き件数 `tagged`」**です（`ratesFrom(counts, denom)`）。PR 文が無い物件は母数から除外します。
- タグは「PR 文に明示的な根拠があるときのみ true」を数えます（`accumulateTags`）。言及が無ければ false 扱いのため、
  タグ率は実態というより**「その特徴をどれだけ積極的に訴求しているか」**に近い指標です。
- 改修要否のみ、タグ率（required / done）とは別に、4 区分（required / done / as_is / unknown）の内訳も集計します。

## 3. 価格の中央値

- **売買物件のみ**（`deal_type === "sale"` かつ `price_yen > 0`）を対象に、配列をソートして中央値を取ります（`median()`）。
- 偶数個のときは中央 2 点の平均を四捨五入。対象が無ければ `null`。賃貸価格は含めません。

## 4. ビニング（ヒストグラム）

固定境界でビン分けします。各バンドは半開区間 `[from, to)`。

- **築年**: `〜1950 / 1951〜1970 / 1971〜1980 / 1981〜1990 / 1991〜2000 / 2001〜2010 / 2011〜`
- **価格**: `〜100万円 / 100〜300万円 / 300〜500万円 / 500〜1000万円 / 1000〜2000万円 / 2000万円〜`

`toHistogram()` は**出現しなかったバンドも 0 件で残します**。自治体間・ビュー間で軸を揃え、グラフ比較を安定させるためです。

## 5. 規模バンド（人口の代理指標）

登録総数 `total` を 5 段階に区分します（`scaleBandOf`、降順に閾値判定）。

| バンド | 条件 |
| --- | --- |
| XL | 200 件以上 |
| L | 100〜199 件 |
| M | 30〜99 件 |
| S | 10〜29 件 |
| XS | 9 件以下 |

> **人口データが無いための代理指標**であり、人口規模の比較ではありません。

## 6.「似たような自治体」の判定

「似ている」は 2 通りの方法で扱います。

### 6-1. 比較の起点となる近接代理（同規模・同県）

- **同規模**: `scale_band` が一致する自治体群。
  - 詳細ページの参考表示（`MunicipalityPage` の `peers.sameBand`）、比較ビューの「同規模を追加」（`addSameBand`、登録総数の降順で追加）。
- **同県**: `prefecture` が一致する自治体群（`addSamePrefecture`）。緯度経度が無く地理的距離を測れないための代理です。
- **同規模平均**: 対象バンドの各自治体の**タグ率を単純平均**します（自治体を 1 票とする非加重平均、`averageTagRates`）。
  大きい自治体に引っ張られない「自治体平均」になります。

### 6-2. 特徴が似た自治体（特徴ベクトル＋コサイン類似度）

詳細ページの「特徴が似た自治体」は、**規模や県によらず特徴ベクトルの近さ**で算出します（`attachSimilar` in `aggregate-core.ts`）。

- 各自治体を次の特徴で 0〜1 主体のベクトルに落とします:
  - 取引種別の構成比（sale / rent）
  - 用途の構成比（residential / commercial / land）
  - PR文タグ率（12 軸）
  - 築年ヒストグラムの構成比（7 バンド）
  - 価格帯ヒストグラムの構成比（6 バンド）
  - 成約率
- 全ペアで**コサイン類似度**を求め、自分を除く**上位 5 件**を提示します（同点は `id` 昇順で決定的）。
- ビルド時に算出し、`municipalities/<id>.json` の `similar`（`[{ id, prefecture, city, score }]`）に格納します。

> 現状はブロック重み付けをしない素朴なコサイン類似度です。次元標準化・重み付け・クラスタリングは今後の改善余地です（[発展余地](#発展余地)）。

## 7. 都道府県ロールアップ（同県平均の基準値）

- 件数系（registered / closed / total / tagged）は単純合算。
- タグ率は**いったん率を件数に戻して合算してから、再び `/ tagged`** で求めます
  （`tagCounts[k] += round(rate * tagged)`）。これは「自治体平均」ではなく**県全体の物件母数での率**で、
  規模の大きい自治体ほど寄与が大きくなります（§6 の同規模平均=非加重とは性質が異なる点に注意）。
- 価格は各自治体の中央値を集めた**「中央値の中央値」**（近似値）です。

## 8. 成約プロファイル（成約した「型」）

- 成約物件だけで取引種別・用途・築年を再集計します。
- **多い PR タグ Top6**: 成約タグ付きの中でカウント降順に上位 6 件（`closedTagCounts` → 件数降順 → `slice(0, 6)`）。
- 成約額・成約日は高欠損のため使わず、**構成（型）だけ**を見ます。
